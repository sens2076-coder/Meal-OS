## §1. 대상 프로필 및 제약 조건
================================================================================

### 1-A. 첫째 (29개월 유아식)
- 식사 구성: 밥(콩 혼합) + 단백질 1 + 채소반찬 3종 (슬롯 고정)
- **밥 조리법**: 백미 + 완두콩(또는 서리태) 1:0.1 비율 혼합 취사 → 매끼 고정
  - 효과: 철분·아연 자연 보완, 씹기 훈련 병행
- **영양 기준** (WHO + 한국영양학회, 29개월 기준):

  | 영양소 | 일일 목표 | 주요 공급원 |
  |--------|-----------|------------|
  | 단백질 | 15~20g    | 소고기 밀프랩 (점심) + 저녁 단백질 |
  | 철분   | 7~8mg     | 소고기 + 철분 씨리얼 (간식 고정) |
  | 칼슘   | 500~700mg | 요거트 + 치즈 + 두부 |
  | 아연   | 3mg       | 소고기 + 콩 + 호박씨 |
  | 비타민A| 300~400μg | 당근 + 청경채 + 애호박 |
  | 비타민C| 35~40mg   | 파프리카(생) + 브로콜리 |
  | 오메가3| 0.7~1.0g  | 생선(주2회) + 멸치볶음(격주) |

- **수율 방어 시스템 (v2.2 통일 기준)**:
  - 모든 소고기 밀프랩: **생육 기준 1포션 = 50g** 일괄 적용
  - 조리 후 수분 손실 감안, 섭취량 약 35~40g 도달
  - ※ 조리법별 수율 차이(완자 70% / 볶음 85%)는 레시피 메모로 안내하되, 포션 기준은 50g으로 통일하여 현장 혼선 방지

- **고정 간식** (영양 계산 제외):
  - 아침: 비타민D 400IU
  - 철분 씨리얼 10g + 요거트 40g

### 1-B. 둘째 (4개월 → 6개월 이유식 준비)
- **이유식 큐브 파이프라인**: 첫째 반찬 조리 중 → 간 넣기 전 소량 분리 → 퓨레/큐브 15ml 냉동
- 🍼 기호 표시 재료: 반드시 큐브 연계 지시어 출력
- 분리 타이밍: 간 투입 직전 (아래 §4 반찬 풀 참조)

## §2. 핵심 운영 원칙 (Core Business Logic)
================================================================================

### 2-1. 식단 사이클
- **토요일 시작 ~ 금요일 종료** (7일 1사이클)

### 2-2. 밀프랩 2배수 원칙
- 모든 점심 소고기 메뉴(라구 포함): **이번 주 사용분 + 다음 주 이월분 = 2배수 제조**
- 소분 방식: 50g × 7포 (이번 주) + 50g × 7포 (다음 주 이월) = 총 14포
- 라구 소스: 1회분 사용(화요일 점심) + 1회분 냉동 이월

### 2-3. 주 3회 조리 리듬 (3일 소진 룰)

| 조리일 | 내용 | 소진 기간 |
|--------|------|----------|
| **토 새벽** | 밀프랩 4종 × 2포션 + 채수·베이스 + 토요일 반찬 퀵조리 | 이번 주 전체 |
| **일 오전** | 반찬 4종 | 일·월·화 (3일) |
| **수 오전** | 반찬 4종 추가 | 수·목·금 (3일) |

**토요일 새벽 조리 용량 주의사항**:
- 소고기 총량 = 밀프랩 4종 × 2배수 → 약 800~900g
- 팬 1개로 4종 순차 조리 (또는 팬 2개 동시 진행)
- 권장 순서: ① 채수 먼저 올리기 → ② 볶음류 → ③ 완자류 → ④ 라구소스
- 예상 총 조리 시간: 약 90~120분

**수요일 반찬 간소화 룰**:
- 4종 중 최소 1종은 **비가열 조리** 배치
- 비가열 대상: 🥢 스틱 슬롯 (항상 생물/비가열) + 그외채소 슬롯 중 1종 (무침·절임류)
- 이유: 주중 부모 피로도 방지

### 2-4. 해동 트리거 자동계산 로직 ← v2.2 NEW

**원칙**: 냉동 이월 밀프랩은 사용 전날 밤 냉장 이동 (자연 해동, 약 8~12시간)

**자동 계산 규칙**:
```
[해동 트리거 테이블]
사용 요일  | 해동 지시 요일 | 해동 지시 시점
---------- | -------------- | --------------
토 (이번주)| 금요일 밤       | 전주 금 22:00
일         | 토요일 밤       | 토 22:00
월         | 일요일 밤       | 일 22:00
화 (라구)  | 월요일 밤       | 월 22:00
수         | 화요일 밤       | 화 22:00
목         | 수요일 밤       | 수 22:00
금         | 목요일 밤       | 목 22:00
```

**출력 규칙**:
- 프린트용 레시피북: 각 요일 상단에 "[전날 밤 해동 알람 🔔]" 뱃지 자동 삽입
- 웹 대시보드: 해당 요일 카드에 해동 알림 아이콘 표시

## §5. 데이터 처리 알고리즘 (Pipeline)
================================================================================

```
STEP 1. Context Load
  → 지난 3주 사용 메뉴 (§8 Weekly Retrospective에서 자동 로드)
  → 현재 재고 잔여물 확인
  → 이월된 밀프랩 종류·수량 확인

STEP 2. Meal Prep Select
  → 전단지 단가 반영하여 §4-A에서 밀프랩 3종 선택 (지난 3주 중복 제외)
  → F01 라구 소스 1종 고정 추가 → 총 4종
  → 저녁 단백질 선택 (§3-B 로테이션 준수)
  → 생선 요일 이번 주 배치 결정 (분산 옵션 선택) + §4-C에서 2가지 조리법 선택

STEP 3. Slot Assignment
  → §4-B 컬러 팔레트 룰 준수하여 반찬 3슬롯 배치
  → 고기=잎채소 페어링 검증 (소·돼지 저녁일 경우 🌿 필수)
  → 일 오전 조리 4종 (일·월·화 소진) vs 수 오전 조리 4종 (수·목·금 소진) 분리
  → 수요일 반찬: 비가열 1종 이상 배치 확인

STEP 4. Quantity Calc
  → 소고기 총량: 밀프랩 3종 × 50g × 14포 + 라구용 = 계산
  → 대용량 식재료 3가지 이상 용도 분산 확인
  → 밥+콩 비율 계산 (1주일 기준)
  → 장보기 수량 최종 확정

STEP 5. Baby Food Tag
  → 🍼 재료 추출
  → 각 재료별 큐브 분리 타이밍·분량 명시 (15ml 기준)

STEP 6. Thaw Schedule (해동 일정 자동 생성)
  → §2-4 해동 트리거 테이블 기반
  → 요일별 해동 대상 밀프랩 자동 매핑
  → 프린트용 [🔔 해동 알람] 뱃지 생성

STEP 7. Value Add
  → 영양 밸런스 검증 (§1-A 기준 7개 항목)
  → Smart Backup 시나리오 생성 (§6 참조)

STEP 8. View Render
  → 웹 대시보드용 JSON 출력 (§9 스키마 기준)
  → 프린트용 흑백 PDF 출력
```

## §6. Smart Backup (잔반·거부 대처 솔루션)

### 거부 시나리오별 대처법

| 거부 상황 | 영양 영향 | 즉시 대처 | 오후 간식 보충 |
|----------|----------|----------|--------------|
| 소고기 메인 거부 | 단백질·철분 부족 | 국물이라도 먹이기 | 치즈 1장 + 두유 100ml |
| 채소 스틱 전체 거부 | 비타민C·식이섬유 부족 | 큼직한 과일 제공 (씹기 훈련 대체) | 방울토마토 5알 or 오이 3조각 |
| 잎채소 반찬 거부 | 비타민A·K 부족 | 국에 잘게 넣어 재시도 | 고구마 30g or 단호박 30g |
| 그외채소 전체 거부 | 식이섬유·아연 부족 | 밥+콩으로 부분 보완 | 바나나 반 개 |
| 밥 거부 | 탄수화물·칼로리 부족 | 밥+국 비벼서 제공 | 쌀과자 2장 + 치즈 1장 |
| 생선 거부 | 오메가3 부족 | 국물·소스만 먹이기 | 호두 2알 or 아마씨 오일 0.5ml |
| 거의 전체 거부 | 종합 부족 | 좋아하는 반찬 1가지만 집중 | 요거트 80g + 과일 |
| 나또 거부 (F02) | — | 달걀찜으로 즉시 대체 | — |

### 영양 주간 경보 시스템
- **🟢 정상**: 7개 영양소 모두 80% 이상 충족
- **🟡 주의**: 1~2개 항목 60~80% → Smart Backup 해당 항목 강조 표시
- **🔴 경보**: 3개 이상 항목 60% 미만 → 다음 주 메뉴 자동 조정 권고 출력

## §8. Weekly Retrospective Loop (Context 피드백) ← v2.2 NEW
================================================================================

### 목적
매주 종료 후 사용 데이터를 기록하여 → 다음 주 §5 STEP 1 Context로 자동 피드백

### 기록 양식 (금요일 저녁 or 토요일 새벽 전)

```yaml
week: "2026-W11"
milprep_used:
  - name: "소불고기"
    rating: 3          # 1~5 (아이 반응)
    leftover: false
  - name: "소고기 파프리카볶음"
    rating: 4
    leftover: false
  - name: "소고기 두부완자"
    rating: 5
    leftover: false

dinner_protein:
  - day: "목"
    menu: "생선 리조또 (가자미)"
    rating: 4
  - day: "금"
    menu: "생선 달걀찜 (연어)"
    rating: 3

side_slots:
  leaf: ["시금치나물", "청경채볶음", "오이무침"]
  stick: ["파프리카(생)", "브로콜리찜", "당근(생)"]
  other: ["애호박볶음", "감자조림", "멸치볶음"]

smart_backup_triggered:
  - day: "화"
    situation: "채소 스틱 거부"
    action: "방울토마토 제공"
    result: "성공"

thawed_carryover:
  - "소고기 라구 소스 (이월분)"

notes: "수요일 아이 피로 → 간소 메뉴 효과적"
```

### 자동 피드백 규칙
- rating ≤ 2 → 다음 4주간 해당 메뉴 제외 풀에 자동 추가
- rating ≥ 4 → 격주 재등장 허용
- leftover: true 연속 2회 → 포션 크기 재검토 알림 출력
