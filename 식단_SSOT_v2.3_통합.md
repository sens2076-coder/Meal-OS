## §4. A/B버전 이중 설계 (어린이집 연동)
================================================================================

### 판단 기준
매일 올라오는 **어린이집 급식 사진** 보고 당일 결정

```
🔴 A버전 (단백질 보충형)
   - 소고기 소량·장식용으로 나온 경우
   - 채소·탄수화물 위주 식단
   - 아이가 점심을 거의 안 먹은 경우

🟢 B버전 (균형형)
   - 닭고기·생선·달걀 등 단백질 충분한 경우
   - 아이가 점심을 잘 먹은 경우
```

### 버전별 구성 차이

| 구분 | A버전 (보충형) | B버전 (균형형) |
|------|--------------|--------------|
| 단백질 포션 | 50g (풀 포션) | 30~35g (감량) |
| 반찬 | 잎채소+스틱+그외채소 | 잎채소+스틱 (2종 간소화 가능) |

## §5. 밀프랩 제조 시스템
================================================================================

### 5-A. 조리 일정

**토요일 (소고기 중심)**
```
① 채수 + 볶음베이스 (공통 기반)
② 소고기 밀프랩 3종 × 2배수
③ 라구 소스 × 2배수
④ 토요일 반찬 퀵조리
```

**일요일 (나머지 단백질)**
```
① 생선 밀프랩 2종 × 2배수
② 기타단백질 2끼분 × 2배수
③ 주간 반찬 4종 (일·월·화 소진용)
```

**수요일 오전**
```
① 반찬 4종 추가 (수·목·금 소진용)
   - 최소 1종 비가열 배치 (부모 피로도 방지)
② 성인 B메뉴 점심 대량 조리 (동시 진행)
```

### 5-B. 포션 계산 (전체 가족)

### 동작 원리
```
항상 최근 4주 파일만 유지
W1 W2 W3 W4 → W5 생성 시 W1 자동 삭제
W2 W3 W4 W5 → W6 생성 시 W2 자동 삭제
```

### 파일 저장 위치
```
/meal-history/
  ├── 2026-W11.json
  ├── 2026-W12.json
  ├── 2026-W13.json
  └── 2026-W14.json  ← 최대 4개 유지
```

### 중복 제거 규칙
| 항목 | 제외 기간 |
|------|---------|
| 소고기 밀프랩 종류 | 최근 3주 사용 메뉴 제외 |
| 기타단백질 종류 | 최근 2주 사용 메뉴 제외 |
| 생선 어종 | 직전 주와 다른 어종 |
| 반찬 슬롯 메뉴 | 최근 2주 사용 메뉴 제외 |
| 성인 점심 메뉴 | 최근 2주 사용 메뉴 제외 |
| 아이 rating ≤ 2 | 4주간 제외 |

### 매주 AI 식단 생성 프로세스
```
STEP 1. 최근 4주 JSON 파일 로드
STEP 2. 중복·제외 메뉴 목록 추출
STEP 3. 전단지 할인 정보 반영
STEP 4. 9끼 배치 확정 (§3 로테이션 준수)
STEP 5. 성인 점심 A·B 메뉴 선택
STEP 6. 반찬 슬롯 배치 (컬러 팔레트 룰)
STEP 7. 해동 일정 자동 계산
STEP 8. 이번 주 JSON 파일 생성·저장
STEP 9. 4주 초과 파일 삭제
```
